## Глава 1. Обзор области и постановка задачи, около 20 страниц

### 1.1. Прикладной и технический контекст

- Дыхательные звуки как тип аудиосигнала.
- Актуальность задачи:
  - дыхательные записи можно получать относительно просто, без сложного измерительного оборудования;
  - один и тот же тип данных подходит для массового сбора, повторных измерений и последующего автоматического анализа;
  - ручной анализ таких записей плохо масштабируется и зависит от качества записи, опыта специалиста и единообразия разметки;
  - автоматическая классификация аудиосегментов может использоваться как вспомогательный инструмент для предварительной сортировки и анализа больших наборов дыхательных записей.
- Прикладная постановка:
  - на вход подается короткий фрагмент дыхательной аудиозаписи;
  - на выходе модель должна выдать класс звукового события или состояния сегмента по разметке датасета;
  - результат работы модели интерпретируется как классификация аудиофрагмента, а не как самостоятельное медицинское заключение;
  - основной практический интерес состоит в построении воспроизводимого конвейера от сырого аудио до метрик качества.
- Особенности записи: шум, разные устройства, короткие фрагменты.
- Основные классы разметки: `normal`, `crackle`, `wheeze`, `both`.
- Техническая сложность задачи:
  - информативные события могут занимать малую часть записи;
  - классы могут быть похожи между собой по спектральным и временным признакам;
  - записи отличаются по громкости, длительности, частоте дискретизации и уровню фонового шума;
  - распределение классов несбалансировано, поэтому простая accuracy не отражает реальное качество модели.
- Почему задача интересна для методов представления аудио и переноса обучения.
- Почему задача подходит для исследования современных моделей:
  - можно сравнить ручные признаки, CNN/CRNN и transformer-подход в одном preprocessing pipeline;
  - предварительно обученные аудиоэнкодеры позволяют проверить, насколько общие аудиопризнаки переносятся на дыхательные записи;
  - ограничение на размер модели до `100M` параметров делает выбор архитектуры практическим.

## Глава 2. Теоретическое описание выбранного решения AST-Base, 15-20 страниц

### 2.1. Общий конвейер AST-Base

6. Классификация через AST-Base.

Обязательный рисунок:

- схема полного pipeline: `WAV -> сегмент -> fixed length -> log-mel -> AST-Base -> class`.

### 2.2. Преобразование аудиосигнала в log-mel

Описать:

- mono loading;
- resampling;
- fixed duration;
- cyclic padding/cropping;
- STFT;
- mel scale;
- log transform;
- z-score normalization.

Дать формулы:

```text
x_tilde = FixLength(x, 8 * 16000)
S(m,k) = sum x[n + mh] w[n] exp(-j 2 pi k n / N)

```

Обязательный рисунок:

### 2.3. Patch embedding и входная последовательность AST

Описать:

- спектрограмма рассматривается как двумерная сетка;
- сетка разбивается на патчи;
- каждый патч разворачивается и проецируется в embedding;
- добавляются class token и positional embedding.

Дать формулы:

```text
z_i = W_p vec(P_i) + b_p
Z_0 = [z_cls; z_1; ...; z_N] + E_pos
```

Обязательный рисунок:

- схема разбиения spectrogram на patches и формирования sequence tokens.

### 2.4. Multi-head self-attention и transformer encoder block

Описать:

- построение `Q`, `K`, `V`;
- scaled dot-product attention;
- multi-head self-attention;
- LayerNorm;
- residual connections;
- MLP block.

Дать формулы:

```text
Q = Z W_Q, K = Z W_K, V = Z W_V
Attention(Q,K,V) = softmax(QK^T / sqrt(d))V
Z'_l = Z_{l-1} + MHSA(LN(Z_{l-1}))
Z_l = Z'_l + MLP(LN(Z'_l))
```

Обязательный рисунок:

- схема transformer encoder block.

### 2.5. Классификационная голова и перенос обучения

Описать:

- используется представление `z_cls` после последнего блока;
- classifier head заменяет исходную AudioSet head на 4 класса;
- модель загружается из pretrained AST checkpoint;
- `freeze_backbone: false`, то есть выполняется full fine-tuning;
- почему обучение с нуля хуже для ограниченного дыхательного датасета;
- почему `AST-Base` укладывается в лимит `100M` параметров: фактически около `86м` параметров.

Дать формулы:

```text
o = W_c LN(z_cls^L) + b_c
p_hat = softmax(o)
```

### 2.6. Функция потерь и учет дисбаланса

Описать:

- train class distribution;
- weighted sampler;
- weighted cross entropy;
- почему это важно для `wheeze` и `both`;
- почему macro F1 и per-class recall важнее простой accuracy.

Дать формулу:

```text
L = -1/B sum_i w_{y_i} log p_hat_{i,y_i}
```

Обязательный рисунок:

- схема учета дисбаланса: `class counts -> class weights -> weighted sampler + weighted CE -> AST training`.
